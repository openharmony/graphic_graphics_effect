# 效果工厂（GEEffectFactory）设计文档

## 一、概述

### 1.1 什么是效果工厂？

效果工厂（`GEEffectFactory`）是 Graphics Effect（GE）组件的核心基础设施，提供了统一的效果创建和管理机制。通过工厂模式，实现了：

- **集中注册**：所有效果类型在一个文件中集中注册
- **O(1)查找**：通过数组索引快速查找和创建效果
- **统一接口**：提供统一的效果创建接口
- **无回退**：工厂完全接管效果创建，无 switch-case 回退

### 1.2 设计目标

1. **性能优化**：从 O(n) 的 switch-case 查找优化为 O(1) 的数组索引查找
2. **代码维护**：新增效果只需添加一行注册代码
3. **扩展性**：支持外部动态加载效果
4. **兼容性**：完全兼容现有的参数系统

### 1.3 适用场景

- 需要频繁创建效果的渲染管线
- 支持动态加载外部效果库
- 需要统一管理大量效果类型
- 需要快速查找和创建特定效果

## 二、架构设计

### 2.1 工厂模式架构

```
┌─────────────────────────────────────────────────────────────┐
│                    GEEffectFactory                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Singleton Instance                       │   │
│  │  - GetInstance()                                     │   │
│  │  - CreateFilter/CreateMask/CreateShape               │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Registration Table                       │   │
│  │  - std::array<std::optional<Creator>, MAX>          │   │
│  │  - Register(type, creator)                          │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Creator Functions                       │   │
│  │  - Lambda functions for each effect type            │   │
│  │  - Call GetParams<T>() and make_shared              │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 文件组织

```
graphics_effect/
├── include/core/
│   └── ge_effect_factory.h          # 工厂类定义和注册宏（183行）
├── src/core/
```

**设计原则：**
- 工厂类作为核心基础设施，放在 `core` 目录
- 注册代码分离到独立文件，便于维护和减少编辑冲突
- 头文件引用按字母序排列，符合代码规范
- 注册按功能分组：Filter → Shader → Mask → Shape

## 三、核心接口

### 3.1 工厂类接口

```cpp
namespace OHOS {
namespace GraphicsEffectEngine {

class GEEffectFactory {
public:
    using VisualEffectImplPtr = std::shared_ptr<Rosen::Drawing::GEVisualEffectImpl>;
    using EffectCreator = std::function<std::shared_ptr<Rosen::Drawing::IGEFilterType>(VisualEffectImplPtr)>;
    
    // 单例模式
    static GEEffectFactory& GetInstance();
    
    // 注册接口
    void Register(Rosen::Drawing::GEFilterType type, EffectCreator&& creator);
    
    // 创建效果接口
    std::shared_ptr<Rosen::Drawing::IGEFilterType> Create(VisualEffectImplPtr impl);
    std::shared_ptr<Rosen::GEShader> CreateShader(VisualEffectImplPtr impl);
    std::shared_ptr<Rosen::GEShaderFilter> CreateFilter(VisualEffectImplPtr impl);
    std::shared_ptr<Rosen::Drawing::GEShaderMask> CreateMask(VisualEffectImplPtr impl);
    std::shared_ptr<Rosen::Drawing::GEShaderShape> CreateShape(VisualEffectImplPtr impl);
    
private:
    static constexpr size_t MAX_EFFECTS = static_cast<size_t>(Rosen::Drawing::GEFilterType::MAX);
    GEEffectFactory() = default;
    
    // 注册表（数组，O(1) 查找）
    std::array<std::optional<EffectCreator>, MAX_EFFECTS> creators_;
};

} // namespace GraphicsEffectEngine
} // namespace OHOS
```

### 3.2 注册宏接口

**核心逻辑提取到内部命名空间的模板函数，宏定义精简到≤10行：**

```cpp
namespace OHOS {
namespace GraphicsEffectEngine {
namespace Internal {  // 内部实现命名空间，不对外暴露

// 模板辅助函数（处理核心逻辑）
template<typename FullClassName>
void RegisterEffect(const char* logTag);

template<typename ParamType, GEFilterType EffectType>
void RegisterExternalEffect(const char* logTag);

template<typename ParamType, typename FallbackClass, GEFilterType EffectType>
void RegisterExternalFallbackEffect(const char* logTag);

} // namespace Internal
}
}
// 简化的宏定义（≤10行）
#define GE_REGISTER_IMPL(ClassName, FullClassName) \
    namespace { \
        struct GEEffectRegistrar_##ClassName { \
            GEEffectRegistrar_##ClassName() { Internal::RegisterEffect<FullClassName>(#ClassName); } \
        }; \
        static GEEffectRegistrar_##ClassName g_effectRegistrar_##ClassName; \
    }

#define GE_FACTORY_REGISTER(ClassName) GE_REGISTER_IMPL(ClassName, ::OHOS::Rosen::ClassName)
#define GE_FACTORY_REGISTER_MASK(ClassName) GE_REGISTER_IMPL(ClassName, ::OHOS::Rosen::Drawing::ClassName)
#define GE_FACTORY_REGISTER_SHAPE(ClassName) GE_REGISTER_IMPL(ClassName, ::OHOS::Rosen::Drawing::ClassName)
#define GE_FACTORY_REGISTER_EXTERNAL(EffectType, ParamType) ...
#define GE_FACTORY_REGISTER_EXTERNAL_FALLBACK(EffectType, ParamType, FallbackClass) ...
#define GE_FACTORY_REGISTER_CUSTOM(EffectType, Lambda) ...
```

**设计要点：**
- 模板函数放在 `Internal` 命名空间，明确标识为内部实现
- 核心逻辑提取到模板函数，宏≤10行符合规范
- 内部实现宏命名为 `GE_REGISTER_IMPL`（避免保留标识符）
- `##ClassName`/`#ClassName` 无法加括号（语法限制）
- `logTag` 用于日志输出效果名称，便于定位问题

## 四、使用方法

### 4.1 基础效果注册

**最简单的注册方式（一行代码）：**

```cpp
// src/core/ge_effect_register.cpp

GE_FACTORY_REGISTER(GEGreyShaderFilter)
GE_FACTORY_REGISTER(GEMESABlurShaderFilter)
GE_FACTORY_REGISTER(GEColorGradientShaderFilter)

// Mask效果
GE_FACTORY_REGISTER_MASK(GERippleShaderMask)
GE_FACTORY_REGISTER_MASK(GEDoubleRippleShaderMask)

// Shape效果
GE_FACTORY_REGISTER_SHAPE(GESDFRRectShaderShape)
GE_FACTORY_REGISTER_SHAPE(GESDFUnionOpShaderShape)
```

**原理：**
- 宏自动提取 `ClassName` 和 `ParamType`
- 通过 `GEFilterTypeInfo<ClassName>::ParamType` 获取参数类型
- 通过 `GEFilterParamsTypeInfo<ParamType>::ID` 获取枚举值
- 生成 Lambda 函数调用 `GetParams<T>()` 和 `make_shared`

### 4.2 自定义逻辑注册

**复杂效果需要自定义创建逻辑：**

```cpp
// KAWASE_BLUR: 根据系统属性切换到 MESA_BLUR
GE_FACTORY_REGISTER_CUSTOM(KAWASE_BLUR,
    [](GEEffectFactory::VisualEffectImplPtr ve) -> std::shared_ptr<Rosen::Drawing::IGEFilterType> {
        auto params = ve->template GetParams<Rosen::Drawing::GEKawaseBlurShaderFilterParams>();
        if (!GERender::IsMesablurAllEnabled()) {
            return std::make_shared<Rosen::GEKawaseBlurShaderFilter>(*params);
        }
        // 转换到 MESA_BLUR
        auto mesaParams = std::make_shared<Rosen::Drawing::GEMESABlurShaderFilterParams>();
        mesaParams->radius = params->radius;
        return std::make_shared<Rosen::GEMESABlurShaderFilter>(*mesaParams);
    })

// LINEAR_GRADIENT_BLUR: 条件外部加载
GE_FACTORY_REGISTER_CUSTOM(LINEAR_GRADIENT_BLUR,
    [](GEEffectFactory::VisualEffectImplPtr ve) {
        auto params = ve->template GetParams<Rosen::Drawing::GELinearGradientBlurShaderFilterParams>();
        if (params->isRadiusGradient) {
            // 尝试外部加载
            void* impl = GEExternalDynamicLoader::GetInstance().CreateGEXObjectByType(...);
            if (impl) return std::shared_ptr<IGEFilterType>(static_cast<IGEFilterType*>(impl));
        }
        return std::make_shared<Rosen::GELinearGradientBlurShaderFilter>(*params);
    })
```

### 4.3 外部效果注册

**支持动态加载外部效果库：**

```cpp
// 纯外部加载
GE_FACTORY_REGISTER_EXTERNAL(DOT_MATRIX, Rosen::Drawing::GEDotMatrixShaderParams)

// 外部加载 + 回退
GE_FACTORY_REGISTER_EXTERNAL_FALLBACK(MESA_BLUR,
    Rosen::Drawing::GEMESABlurShaderFilterParams,
    Rosen::GEMESABlurShaderFilter)
```

**外部加载流程：**
1. 通过 `GEExternalDynamicLoader` 加载外部库
2. 如果加载成功，返回外部效果对象
3. 如果加载失败（EXTERNAL_FALLBACK），返回内置效果

### 5.2 渲染管线流程（ge_render.cpp）

工厂直接调用，无回退：

```cpp
std::shared_ptr<GEShaderFilter> GERender::GenerateShaderFilter(
    const std::shared_ptr<Drawing::GEVisualEffect>& vef)
{
    auto ve = vef->GetImpl();
    auto shaderFilter = GEEffectFactory::GetInstance().CreateFilter(ve);
    if (shaderFilter) {
        shaderFilter->SetShaderFilterCanvasinfo(vef->GetCanvasInfo());
        shaderFilter->SetSupportHeadroom(vef->GetSupportHeadroom());
    }
    return shaderFilter;
}

std::shared_ptr<GEShader> GERender::GenerateShaderEffect(
    const std::shared_ptr<Drawing::GEVisualEffectImpl>& ve)
{
    return GEEffectFactory::GetInstance().CreateShader(ve);
}
```

## 七、性能分析

### 7.1 性能对比

| 方式 | 查找复杂度 | 添加新效果 | 维护成本 |
|------|-----------|----------|----------|
| **Switch-case** | O(n) - 线性查找 | 修改多处代码 | 高 - 分散维护 |
| **工厂模式** | O(1) - 数组索引 | 添加一行注册 | 低 - 集中维护 |

### 7.2 性能优势

1. **查找性能**：
   - Switch-case: 线性查找 O(n)，随着效果数量增加性能下降
   - 工厂模式: 数组索引 O(1)，性能稳定不受效果数量影响

2. **内存占用**：
   - 注册表大小固定（基于效果类型数量）
   - 每个 Lambda 函数编译期优化

### 7.3 性能测试建议

```cpp
void PerformanceTest() {
    const int iterations = 10000;
    
    auto start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < iterations; i++) {
        auto effect = GEEffectFactory::GetInstance().CreateFilter(ve);
    }
    auto factoryTime = std::chrono::duration_cast<std::chrono::milliseconds>(
        std::chrono::high_resolution_clock::now() - start).count();
    
    GE_LOGI("Factory time: %dms for %d iterations", factoryTime, iterations);
}
```

## 八、线程安全

### 8.1 单例模式线程安全

工厂使用 Meyers 单例模式，C++11 标准保证线程安全：

```cpp
GEEffectFactory& GEEffectFactory::GetInstance()
{
    static GEEffectFactory instance;  // C++11 保证线程安全初始化
    return instance;
}
```

### 8.2 操作线程安全分析

| 操作 | 线程安全性 | 说明 |
|------|----------|------|
| `GetInstance()` | ✅ 安全 | C++11 保证静态局部变量初始化线程安全 |
| `Register()` | ✅ 安全 | 只在静态初始化阶段调用（程序启动前，单线程） |
| `Create()` | ✅ 安全 | 只读操作，访问 `creators_[]` 数组 |

### 8.3 调用时机保证

**静态初始化顺序：**
1. 程序启动前，静态对象构造
2. `GEEffectRegistrar_##ClassName` 构造函数调用 `Register()`
3. 所有效果在 `main()` 前完成注册

**运行时调用：**
- `Create()` 在渲染线程调用
- 只读已填充的 `creators_[]` 数组
- 无运行时动态注册需求

## 九、扩展机制

### 8.1 外部动态加载

**支持动态加载外部效果库：**

```cpp
// 外部效果加载流程
std::shared_ptr<IGEFilterType> CreateExternalEffect(FilterType type, const void* params) {
    // 1. 通过动态加载器加载
    void* impl = GEExternalDynamicLoader::GetInstance().CreateGEXObjectByType(
        static_cast<uint32_t>(type), 
        sizeof(ParamType), 
        params);
    
    // 2. 转换为智能指针
    if (impl) {
        return std::shared_ptr<IGEFilterType>(static_cast<IGEFilterType*>(impl));
    }
    
    // 3. 失败时回退
    return nullptr;
}
```

### 8.2 外部效果库要求

**外部效果库需要实现：**
- 导出 `CreateGEXObjectByType` 函数
- 支持参数序列化/反序列化
- 符合效果接口规范

### 8.3 扩展注册示例

## 十一、使用建议

### 11.1 效果注册方式选择

1. **优先使用基础宏**
   ```cpp
   // 简单效果：一行注册
   GE_FACTORY_REGISTER(GESimpleFilter)
   ```

2. **复杂逻辑使用自定义宏**
   ```cpp
   // 需要特殊处理的效果
   GE_FACTORY_REGISTER_CUSTOM(COMPLEX_EFFECT, [](auto ve) {
       // 自定义逻辑
   })
   ```

3. **外部效果使用外部宏**
   ```cpp
   // 外部加载效果
   GE_FACTORY_REGISTER_EXTERNAL(EFFECT_NAME, ParamType)
   ```

### 11.2 添加新效果的步骤

1. **确认效果类有 ParamType 定义**
   ```cpp
   // 头文件中检查
   DECLARE_GEFILTER_TYPEFUNC(GENewEffect, Drawing::GENewEffectParams);
   ```

2. **确认参数已注册**
   ```cpp
   // ge_params_reflection.h 中检查
   GE_PARAMS_TYPE_INFO(GENewEffectParams, NEW_EFFECT, NewEffect);
   ```

3. **添加工厂注册**
    ```cpp
    // src/core/ge_effect_register.cpp 中添加
    GE_FACTORY_REGISTER(GENewEffect)
    ```

4. **编译验证**
   ```bash
   hb build graphics_effect -i
   ```

5. **测试验证**
   ```bash
   hb build graphics_effect -t
   ```

### 11.3 常见问题处理

**Q: 注册失败，找不到 ParamType**
```cpp
// 解决：在效果类头文件中添加

```

**Q: 参数未注册**
```cpp
// 解决：在 ge_shader_filter_params.h 中添加
REGISTER_GEFILTERPARAM_TYPEINFO(YOUR_EFFECT, GEYourEffectParams, "YourEffect");
```

**Q: 编译错误：找不到 GetParams**
```cpp
// 解决：检查参数类型是否正确
// 确保 GE_PARAMS_TYPE_INFO 已注册
```
